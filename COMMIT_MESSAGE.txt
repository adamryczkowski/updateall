feat: Initialize update-all Python monorepo for Milestone 1 (MVP)

This commit sets up the complete project structure for the update-all
system - a plugin-driven system update orchestrator for Linux.

## Project Structure

The project is organized as a monorepo with 5 independent Poetry subprojects:

- **core**: Core models, interfaces, and abstractions
- **cli**: Typer-based command-line interface
- **plugins**: Update plugins for apt, pipx, flatpak
- **ui**: Rich-based terminal UI components
- **stats**: Update history and statistics tracking

## Key Features Implemented

### Core Module
- Pydantic models for configuration (PluginConfig, GlobalConfig, SystemConfig)
- Execution result models (ExecutionResult, PluginResult, ExecutionSummary)
- Abstract plugin interface (UpdatePlugin) with lifecycle hooks
- Plugin executor and config loader interfaces

### CLI Module
- Main Typer application with rich markup support
- Commands: run, status, history
- Subcommands: plugins (list, enable, disable), config (show, init)
- Version flag with callback

### Plugins Module
- BasePlugin abstract class with common functionality
- Command execution with timeout handling and sudo support
- Concrete plugins: AptPlugin, PipxPlugin, FlatpakPlugin
- Plugin registry with entry point discovery support

### UI Module
- ProgressDisplay: Live progress tracking with Rich
- ResultsTable: Formatted results display
- SummaryPanel: Update run summary with statistics

### Stats Module
- HistoryStore: JSON-based persistent history storage
- StatsCalculator: Aggregate statistics calculation
- XDG Base Directory Specification compliance
